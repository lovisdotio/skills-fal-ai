## Model Selection Guide

### By Use Case

| Use Case | Recommended Model |
|----------|------------------|
| **Best quality image** | `fal-ai/kling-image/o3/text-to-image` |
| **Fastest image** | `fal-ai/z-image/turbo` |
| **Photorealistic portrait** | `fal-ai/flux-2/klein/9b` |
| **Text in images** | `fal-ai/glm-image` or `fal-ai/ideogram/v3` |

## Tips

1. **Check pricing**: Use `./scripts/models.sh --pricing <endpoint>`
2. **Use LoRA variants**: For custom styles, prefer `-lora` endpoints
3. **Distilled models**: Faster with minimal quality loss
4. **Pro vs Standard**: Pro = better quality, Standard = cost-effective
5. **Flash/Turbo**: Best for previews and iterations
6. **Always use queue**: For video/3D, always use queue endpoints
7. **Set seeds**: For reproducibility across generations
